# MAX VCTeam 소개 및 데모 시나리오

> 목적: 에이벤처스 워크샵에서 MAX VCTeam을 자연스럽게 소개하고, 투자검토 보조 구조를 이해시킨다.

## 1. 한 문장 소개

> MAX VCTeam은 투자심사역의 투자검토를 돕기 위해 워크샵에서 소개하는 VC 업무 특화 AI Native 팀입니다.

## 2. 포지셔닝

MAX VCTeam은 투자 의사결정 자동화 도구가 아니다.

| 오해 | 정확한 설명 |
| --- | --- |
| AI가 투자 여부를 판단한다 | 투자 여부는 사람이 판단한다 |
| IR 자료를 넣으면 결론을 준다 | 투자 포인트, 리스크, 질문, 확인 필요 자료를 구조화한다 |
| 심사역을 대체한다 | 심사역이 더 좋은 질문을 더 빨리 만들도록 돕는다 |
| 모든 자료를 자동 수집한다 | 승인된 자료와 권한 안에서만 작동한다 |

## 3. 대상 업무

| 업무 | MAX VCTeam이 도울 수 있는 부분 |
| --- | --- |
| 딜 소싱 후 1차 검토 | IR Deck 요약, 투자 포인트, 레드플래그 정리 |
| 산업 리서치 | 시장 구조, 경쟁사, 밸류체인, 규제·기술 이슈 정리 |
| 파트너 미팅 준비 | 회사 개요, 핵심 질문, 미팅 아젠다 생성 |
| IC 준비 | 투자검토 메모 초안, 예상 반론, 추가 확인자료 목록 |
| 포트폴리오 모니터링 | 월간 업데이트 요약, 런웨이/매출/채용 이슈 표시 |
| LP 보고 준비 | 보고 초안, 확인 필요 수치, 민감정보 점검 |

## 4. AI Native 팀 구조

```text
MAX VCTeam
  ├─ Deal Intake Agent
  │   └─ IR 자료와 회사 소개 자료를 읽고 기본 정보를 추출
  ├─ Investment Thesis Agent
  │   └─ 투자 매력, 시장성, 팀, 제품, 사업모델을 정리
  ├─ Market Research Agent
  │   └─ 시장, 경쟁사, 밸류체인, 규제 변화를 조사
  ├─ Risk Reviewer
  │   └─ 법률, 회계, 기술, 사업, 데이터 부족 리스크를 표시
  ├─ IC Memo Agent
  │   └─ 투자검토 메모와 질문 목록 초안 작성
  └─ Human Partner
      └─ 최종 판단, 우선순위, 외부 공유 승인
```

## 5. 데모 시나리오

### 데모 입력

가상 스타트업 IR Deck 또는 더미 요약 자료:

```text
회사명: Demo Robotics
분야: 제조 현장용 비전 검사 로봇
단계: Series A
요청: 30억원 투자 유치
핵심 주장: 검사 속도 40% 향상, 불량률 25% 감소, 6개 제조 고객 PoC 진행
우려: 매출 반복성 부족, 고객 집중도 높음, 경쟁사 대비 기술 차별성 검증 필요
```

### 데모 프롬프트

```text
이 자료는 교육용 더미 IR 자료다.
투자 여부를 판단하지 말고, 투자심사역이 1차 검토할 수 있도록 정리해줘.

출력:
1. 회사 개요
2. 투자 매력 포인트
3. 핵심 리스크
4. 추가 확인자료 요청 목록
5. 파트너 미팅 질문 10개
6. IC에서 나올 수 있는 반론 5개
7. 사람이 판단해야 할 항목

근거가 부족한 내용은 "확인 필요"로 표시해줘.
```

### 데모 결과 메시지

```text
이 결과는 투자 결론이 아닙니다.
투자심사역이 더 빠르게 핵심 질문을 만들고,
파트너와 논의할 쟁점을 정리하기 위한 업무 보조 산출물입니다.
```

## 6. MAX Studio 하네스 기반 설명

MAX Studio는 MAX VCTeam을 만드는 제작 환경으로 설명한다.

| 구성요소 | MAX Studio에서 하는 일 |
| --- | --- |
| 업무 목표 | 투자검토 보조, 포트폴리오 업데이트 요약 등 업무 단위 정의 |
| 자료 구조 | IR Deck, 시장 자료, 월간 업데이트, LP 보고 재료 등 입력 폴더 설계 |
| Skill | IR 평가, 시장 리서치, 리스크 리뷰, IC 메모 작성 절차화 |
| Connector | 승인된 자료원과 앱 연결 |
| Reviewer | 투자 판단 제외, 근거 부족, 민감정보, 법률·회계 리스크 검토 |
| Test Log | 샘플 딜별 결과 품질과 오류 기록 |

## 7. 후속 논의

### 제안 문장

```text
워크샵 이후 에이벤처스가 관심을 가지시는 업무가 있다면 별도 논의로 이어갈 수 있습니다.
실제 민감자료가 아니라 비식별 샘플 딜 자료나 공개자료를 기준으로 시작하고,
투자심사역이 실제로 쓸 만한 질문과 메모가 나오는지 함께 검증하겠습니다.
```

### 후속 논의 범위

| 항목 | 범위 |
| --- | --- |
| 기간 | 별도 협의 |
| 자료 | 비식별 IR 자료 또는 공개자료 기반 샘플 |
| 기능 | IR 스크리닝, 리스크 정리, 질문 생성, 메모 초안 |
| 검증 지표 | 소요시간, 질문 품질, 리스크 검토 누락 가능성, 심사역 만족도 |
| 제외 | 실제 투자 결정, 대외 발송, 민감 시스템 연동 |

## 8. 후속 미팅 아젠다

1. 에이벤처스에서 가장 시간이 많이 드는 투자검토 단계 확인
2. 실제 자료를 쓰지 않고 만들 수 있는 더미/비식별 샘플 구조 합의
3. MAX VCTeam 첫 적용 업무 선택
4. 보안·권한·외부 전송 기준 정리
5. 후속 논의 일정과 피드백 방식 협의
