1부. AI Native 기술의 현재와 미래

1. 타이틀

제목:

AI Native 기술의 현재와 미래

부제:

Claude Cowork에서 MAX VCTeam까지, VC 업무를 돕는 AI Native 팀의 구조

2. 오늘의 질문

핵심 문장:

AI에게 어디까지 일을 맡길 수 있고, 어디부터는 사람이 판단해야 하는가?

강조:

  • 오늘은 Claude 메뉴 교육이 아니다.
  • 투자검토 업무를 AI Native 방식으로 바라보는 시간이다.
  • 실습은 Skills, Connectors, Scheduled, Plugins가 실제 업무 구조에서 어떤 의미인지 체감하기 위한 것이다.

3. AI 기술의 변화

도식:

Chatbot → Copilot → Agent → AI Native Team

메시지:

  • 답변 생성에서 업무 실행으로 이동
  • 단일 프롬프트에서 장시간 다단계 작업으로 이동
  • 개인 생산성에서 조직 운영 구조로 이동

4. 모델에서 에이전트로

핵심:

모델 + 실행 환경 + 도구 + 권한 + 검증 = 업무 수행 에이전트

VC 예시:

  • IR Deck 읽기
  • 시장 자료 검색
  • 리스크 질문 생성
  • Word/PPT/Excel 산출물 작성
  • 심사역 승인 요청

5. AI Native 조직

정의:

AI Native 조직은 AI가 일할 수 있도록 업무를 목표, 자료, 절차, 권한, 검증, 승인, 기록으로 설계한 조직이다.

VC 업무 적용:

업무AI가 도울 일사람이 할 일
투자검토요약, 질문, 리스크투자 판단
산업 리서치시장·경쟁 구조화투자 가설 평가
포트폴리오 관리이상 신호 탐지지원 우선순위 결정
LP 보고초안·수치 확인대외 보고 승인

6. 하네스

한 문장:

하네스는 모델을 실제 업무에 안전하게 연결하는 운영 장치다.

구성:

목표
+ 자료
+ Skill
+ Connector
+ Reviewer
+ 사람 승인
+ Log

7. 왜 VC에 하네스가 중요한가

핵심:

  • VC 업무는 정보 비대칭이 크다.
  • IR, 계약, LP, 포트폴리오 자료는 민감하다.
  • AI 결과가 그럴듯해도 투자 판단은 사람이 해야 한다.
  • 따라서 권한, 검증, 승인, 로그가 필요하다.

8. 매직에꼴 MAX 솔루션

메시지:

MAX는 AI Native 업무를 설계하고 실험하기 위한 하네스 제작·운영 솔루션으로 소개한다.

활용 예시:

  • 맞춤형 교육자료 생성
  • 업무 하네스 설계
  • AI Native 팀 데모
  • MAX Studio 기반 업무별 AI 팀 제작

9. AI Native 팀 데모

구성:

Orchestrator
  ├─ Deal Screener
  ├─ Market Researcher
  ├─ Risk Reviewer
  ├─ IC Memo Writer
  └─ Human Partner

메시지:

AI Native 팀은 하나의 만능 AI가 아니라, 역할을 나누고 사람이 판단하는 업무 구조다.

10. MAX VCTeam

정의:

MAX VCTeam은 투자심사역의 투자검토를 돕는 VC 업무 특화 AI Native 팀이다.

기능:

  • IR Deck 1차 스크리닝
  • 투자 포인트와 리스크 분리
  • 시장·경쟁 리서치 보조
  • IC 예상 질문 생성
  • 투자검토 메모 초안
  • 포트폴리오 업데이트 요약

사람이 확인할 항목:

  • 투자 여부 확정
  • 기업가치 판단
  • 법률·회계 판단
  • 민감자료 외부 공유

11. 실습 연결

표:

14. AI가 맡을 일과 사람이 맡을 일

AI가 맡을 일:

  • 자료 요약
  • 질문 생성
  • 리스크 목록화
  • 메모 초안
  • 반복 모니터링

사람이 맡을 일:

  • 투자 판단
  • 외부 공유 승인
  • 법률·회계 판단
  • LP 커뮤니케이션 최종 승인

17. MAX VCTeam 소개와 후속 논의

제안:

비식별 샘플 딜 자료 또는 공개자료를 기준으로 MAX VCTeam의 투자검토 보조 구조를 소개하고, 관심 업무가 있으면 후속 논의로 연결한다.

검증 지표:

  • 검토 시간 절감
  • 질문 품질
  • 리스크 검토 누락 가능성
  • 메모 초안 활용도
  • 심사역 만족도
