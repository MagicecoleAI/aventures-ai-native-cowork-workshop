# 클로징 회고 및 후속 제안 멘트

> 사용 시간: 17:10-17:30  
> 목적: 실습 경험을 AI Native 팀 도입과 MAX VCTeam 파일럿 검토 제안으로 연결한다.

## 1. 회고 질문

참가자에게 5분 정도 질문한다.

```text
오늘 실습한 네 가지를 에이벤처스 업무로 번역하면 무엇일까요?
```

| 오늘 실습 | 업무 의미 |
| --- | --- |
| IR 평가 스킬 | 투자검토 기준을 재사용 가능하게 만든다 |
| Chrome/PlayMCP 커넥터 | AI가 승인된 외부 자료와 앱을 다룰 수 있다 |
| 예약 작업 | 반복 리서치와 모니터링을 정해진 조건으로 실행할 수 있다 |
| Legal Plugin | 직무별 검토 기준을 묶어 문서를 보강할 수 있다 |

추가 질문:

1. 에이벤처스에서 반복성이 가장 높은 투자검토 업무는 무엇인가?
2. AI가 초안으로 도와도 되는 업무와 사람이 반드시 판단해야 하는 업무는 무엇인가?
3. 오늘 실습 중 실제 업무에 가장 가까웠던 장면은 무엇인가?
4. MAX VCTeam이 있다면 어떤 딜 검토 단계부터 맡겨보고 싶은가?

## 2. 마무리 핵심 메시지

```text
오늘의 결론은 "Claude를 잘 쓰자"가 아닙니다.
결론은 "AI가 일할 수 있는 구조를 만들자"입니다.
```

```text
AI Native 팀은 한 명의 만능 AI가 아닙니다.
IR을 읽는 역할, 시장을 조사하는 역할, 리스크를 검토하는 역할, 메모를 쓰는 역할, 그리고 최종 판단을 하는 사람의 조합입니다.
```

```text
VC 업무에서 AI의 가치는 투자 결론을 대신 내리는 데 있지 않습니다.
더 많은 자료를 더 빨리 구조화하고, 놓칠 수 있는 질문을 드러내고, 사람이 판단해야 할 쟁점을 선명하게 만드는 데 있습니다.
```

## 3. 후속 진행 안내

```text
워크샵 이후에는 바로 전사 확대를 추진하기보다 작은 PoC로 시작하는 것이 좋습니다.
첫 단계는 에이벤처스 업무 중 하나를 고르는 것입니다.
IR 1차 검토, 산업 리서치, 포트폴리오 월간 업데이트, LP 보고 초안 중 하나를 선택해도 충분합니다.
```

후속 단계:

| 단계 | 내용 |
| --- | --- |
| 1단계 | 적용할 업무 1개 선택 |
| 2단계 | 실제 민감정보를 제거한 샘플 자료 구성 |
| 3단계 | MAX VCTeam 파일럿 검토 |
| 4단계 | 심사역이 결과 품질과 시간 절감 효과 검토 |
| 5단계 | 보안·권한·운영 책임을 정리해 PoC 여부 판단 |

## 4. 매직에꼴이 도와줄 수 있는 부분

```text
매직에꼴은 단순히 프롬프트를 만들어드리는 역할이 아닙니다.
업무를 AI Native 팀이 수행 가능한 구조로 바꾸고,
자료, 권한, 검토 기준, 승인 흐름, 테스트 방법을 함께 설계하는 역할을 할 수 있습니다.
```

지원 범위:

| 영역 | 지원 내용 |
| --- | --- |
| 업무 발굴 | AI 적용 가치가 있는 VC 업무 장면 선정 |
| 하네스 설계 | Skill, Connector, Reviewer, Test Log 구조 설계 |
| 자료 비식별화 | 실습과 PoC에 쓸 더미/비식별 자료 구조화 |
| MAX Studio | 업무별 AI Native 팀 제작과 테스트 |
| MAX VCTeam | 투자검토 보조 AI Native 팀 파일럿 검토안 제안 |
| PoC 설계 | 기간, 지표, 보안, 권한, 운영 책임 정리 |

## 5. MAX VCTeam 파일럿 검토 소개

```text
MAX VCTeam은 에이벤처스 같은 VC의 투자검토 업무를 돕기 위한 AI Native 팀입니다.
IR Deck을 읽고, 투자 포인트와 리스크를 분리하고,
산업 리서치와 IC 질문을 만들고,
투자검토 메모 초안을 작성하는 데 초점을 둡니다.
```

```text
다시 강조드리지만 MAX VCTeam은 투자 판단을 자동화하지 않습니다.
투자심사역이 더 빠르게, 더 일관되게, 더 좋은 질문을 만들도록 돕는 업무 보조 팀입니다.
```

제안:

```text
에이벤처스가 원하시면 워크샵 이후 협의를 거쳐 2-4주 MAX VCTeam 파일럿 검토안을 제안드릴 수 있습니다.
비식별 샘플 딜이나 공개자료를 기준으로 시작하고,
실제 심사역 관점에서 결과가 쓸 만한지 함께 검토하겠습니다.
```

## 6. 최종 클로징 멘트

```text
오늘 여러분이 실습한 것은 단순한 기능 네 가지가 아닙니다.
Skill은 업무 기준을, Connector는 자료와 앱 연결을, Scheduled는 반복 실행을, Plugin은 직무별 AI 팀 구성을 보여줬습니다.

이 네 가지가 합쳐지면 AI Native 팀이 됩니다.
그리고 VC 업무에 맞게 구성하면 MAX VCTeam 같은 투자검토 보조 팀이 됩니다.

에이벤처스의 다음 단계는 거창한 AI 전환이 아니라,
투자검토 업무 하나를 골라 작게 검증하는 것입니다.
매직에꼴은 그 과정을 함께 설계하고, MAX VCTeam을 통해 실제 활용 가능성을 검증할 수 있습니다.

감사합니다.
```
